# Session Memory Skeleton

Für jede Feature-, Repair-, Integrations- oder Architektur-Session:

1. `SESSION_TEMPLATE.md` nach `YYYYMMDD_<thema>.md` kopieren;
2. Auftrag, Evidenz, Scope, Änderungen und Verifikation festhalten;
3. niemals Secrets oder produktive Datenwerte aufnehmen;
4. Session-Notizen nicht ohne Prüfung als dauerhafte Wahrheit behandeln;
5. dauerhafte Fakten nach `../MEMORY.md`, Entscheidungen nach `../DECISIONS.md` und Code-Landkarten nach `../ARCHITECTURE.md` überführen;
6. `../SESSION_HANDOFF.md` aktualisieren;
7. Akzeptanz ausschließlich über externe Review und `../ACCEPTANCE_LOG.md` dokumentieren.
